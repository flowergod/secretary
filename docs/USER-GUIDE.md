## 功能概览

Project Secretary 是一个基于 AI 的个人助理，可以帮助你：

- ✅ **任务管理** - 创建、修改、删除、完成任务
- 📅 **日程管理** - 安排会议、设置提醒、查询日程
- 📋 **项目管理** - 创建项目、追踪进度、管理子任务
- 🔄 **循环任务** - 固定间隔循环、基于完成的循环
- 🧠 **智能拓展** - AI 自动建议后续步骤
- 🔍 **信息检索** - 语义搜索历史记录

## 快速开始

### 安装

```bash
# 克隆项目
git clone https://github.com/flowergod/secretary.git
cd secretary

# 安装依赖
npm install

# 配置文件
cp config.example.yaml config.yaml
# 编辑 config.yaml 填入你的飞书配置

# 构建
npm run build
```

### 首次使用

与 AI 助理对话非常简单，只需说出你的需求：

```
用户：帮我安排下周三下午3点开会
AI：✓ 已添加日程「开会」
    - 日期：2026-05-06
    - 时间：15:00
    - 优先级：中
    - 状态：待执行

用户：今天有什么安排？
AI：今天共有3件事：
    - 09:00 团队周会
    - 14:00 方案评审
    - 19:00 牙医预约
```

```markdown

# AI 将：
# 1. 确认要删除的任务
# 2. 从飞书表格中删除
# 3. 从 iCloud 日历中删除（如有）
```

## 循环任务

### 固定间隔循环

```markdown
用户：每两周提醒我做一次体检

# AI 将创建：
# - 类型：任务
# - 标题：体检
# - 循环类型：每两周
# - 下次提醒：2周后
```

支持的循环模式：
- 每小时 / 每分钟
- 每天
- 每周一 / 每周五 / 每周
- 每月
- 每半年 / 每年

### 基于完成的循环

```markdown
用户：看完医生后3个月提醒复查

# AI 将创建：
# - 类型：任务
# - 标题：复查
# - 循环类型：完成后N天
# - 规则：完成后90天提醒
```

### 自动续接

当循环任务完成后，AI 会自动创建下一个周期的任务：

```
1. 用户完成「体检」任务
2. AI 检测到这是循环任务
3. AI 自动创建下一个「体检」任务（2周后）
```

## 智能拓展

当创建某些任务时，AI 会主动询问是否需要规划后续步骤。

### 自动触发场景

| 关键词 | 拓展类型 | 说明 |
|--------|---------|------|
| 医生/体检/复查 | 医疗复查 | 3个月后提醒复查 |
| 需求/产品 | 开发规划 | AI 生成开发步骤 |

### 使用示例

```markdown
用户：完成需求后自动规划开发

# AI 将：
# 1. 创建任务「完成需求」
# 2. 标记 needs_expansion = true
# 3. 询问：「需求完成后，是否需要我帮你规划开发进度？」

用户：好的

# AI 调用 AI 模型生成开发计划：
# - 完成需求 (高优, 2天)
# - 技术方案设计 (高优, 3天)
# - 核心功能开发 (高优, 5天)
# - 单元测试编写 (中优, 2天)
```

## 配置说明

### 配置文件结构

```yaml
# config.yaml
feishu:
  appId: "cli_xxxxxxxx"        # 飞书应用 ID
  appSecret: "xxxxxxxx"        # 飞书应用密钥
  tableToken: "xxxxxxxx"        # 飞书表格 Token
  tableId: "xxxxxxxx"           # 飞书表格 ID
  webhookUrl: "https://..."     # 飞书 Webhook（可选）

icloud:
  appleId: "your@email.com"      # Apple ID
  appPassword: "xxxx-xxxx"      # 应用专用密码

ai:
  primary:
    provider: "volcano"         # 主 AI：volcano 或 minimax
    apiKey: "${VOLCANO_API_KEY}"
    baseUrl: "https://ark.cn-beijing.volces.com/api/v3"
  fallback:
    provider: "minimax"         # 备 AI
    apiKey: "${MINIMAX_API_KEY}"

reminders:
  morning:
    enabled: true
    time: "8:30"
  evening:
    enabled: true
    time: "21:00"
  weekendSummary:
    enabled: true
    time: "18:00"
    dayOfWeek: 5
  preEvent:
    enabled: true
    minutesBefore: 15
```

### 环境变量

敏感信息可通过环境变量注入：

```bash
export VOLCANO_API_KEY="your-volcano-key"
export MINIMAX_API_KEY="your-minimax-key"
export FEISHU_APP_ID="your-feishu-app-id"
export FEISHU_APP_SECRET="your-feishu-app-secret"
```

### 飞书配置

1. 创建飞书应用：https://open.feishu.cn/app
2. 获取 App ID 和 App Secret
3. 开通「多维表格」权限
4. 获取表格的 App Token 和 Table ID

### iCloud 配置

1. 登录 Apple ID：https://appleid.apple.com
2. 创建应用专用密码
3. 开启日历共享权限

## 常见问题

### Q: 任务创建后多久会同步到飞书？

A: 实时同步。创建/更新/删除操作会立即同步到飞书表格。

### Q: iCloud 日历同步失败怎么办？

A: 检查：
1. Apple ID 和应用专用密码是否正确
2. 日历分享是否开启
3. 网络连接是否正常

### Q: 如何删除循环任务？

A: 告诉 AI「删除任务XXX」，AI 会删除当前任务并询问是否继续删除后续循环。

### Q: AI 模型调用失败怎么办？

A: 系统采用双链路设计：
1. 主链路：火山方舟
2. 备链路：MiniMax

主链路失败时自动切换到备链路。

### Q: 支持哪些语言？

A: 目前主要支持中文输入。英文输入也可以识别但响应为中文。
